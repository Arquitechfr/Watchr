# Audit UI/UX — Modernisation de l'app mobile/web Watchr

Audit des écrans, composants et patterns UX, avec priorisation par impact.

---

## 1. Bottom Tabs & Navigation

### `MainTabs.tsx` — Tab bar
- ✅ **Haptic feedback** sur tab press (`hapticSelection` via `SegmentedControl`)
- **Pas d'animation** de transition entre tabs (pas d'indicateur animé type `react-native-tab-view`)
- **SearchTabIcon (FAB)** : shadow trop prononcé (`shadowOpacity: 0.4`), pourrait utiliser `shadowColor: #000`
- **NotificationBadge** : `minWidth: 20` pourrait être dynamique selon le chiffre
- **tabBarLabelStyle** : pas de variation font family (Outfit)
- **Pas de badge "new"** sur les tabs Series/Movies quand nouveaux épisodes disponibles

### `DesktopSidebar.tsx` — Sidebar web
- **Pas de hover states** sur les items (`hover:bg-surface` manquant)
- **Pas d'indicateur animé** de tab actif (ligne gauche ou barre)
- **Pas de collapse/reduce** mode pour la sidebar
- **Logo** : utilise `Ionicons play-circle` au lieu du vrai logo Watchr (incohérent avec `MainHeader`)
- **Pas de section "Quick Actions"** (import, export, settings) en bas de sidebar
- **Largeur fixe 220px** — pas de resizable

### `RootNavigator.tsx` — Stack navigation
- **Pas de transitions personnalisées** entre écrans (default native uniquement)
- **Pas de shared element transitions** sur les posters → détail
- **Pas de deep linking** vers les sous-pages profile (EditProfile, Library, etc.)

## 2. Écrans principaux (Bottom Tabs)

### `SeriesScreen.tsx` (567 lignes — dépasse la limite de 300)
- **Problème architecture** : fichier trop long, extraire `UnwatchedList` et `UpcomingList` dans des composants séparés
- ✅ **TopTabs** : utilise désormais `SegmentedControl` unifié
- **SearchBar toggle** : apparition/disparition sans animation
- **Skeleton** : pas de shimmer
- **Bouton "View All"** : plain `TouchableOpacity` en bas, non sticky, non flottant
- ✅ **UnwatchedEpisodeRow** : swipe-to-mark-watched implémenté (swipe gauche = watched, droite = détail)
- ✅ **Long-press** : actions rapides via `QuickActionMenu` sur `PosterCard` et `MediaRow`
- **Pas de tri** des épisodes (par date, par show, par saison)
- **Grid mode** : colonnes fixes, pas de staggered/masonry
- ✅ **Empty state** : `EmptyState` redessiné avec pulse animé et circular background
- **ImportProgressBanner** : placement entre header et contenu peut pousser le tabs vers le bas

### `MoviesScreen.tsx` (412 lignes — dépasse la limite de 300)
- ✅ **MovieCategoryTabs** : utilise désormais `SegmentedControl` unifié
- **MovieCard** : pas de badge visuel de status, pas de progress bar pour les films "in progress"
- **Filters** : seulement visibles quand search est toggled — pas accessible indépendamment
- **Pas de tri** (par titre, par année, par note, par date d'ajout)
- **Pas de groupement** (par genre, par année, par status)
- **Grid mode** : OK avec PosterCard mais pas de overlay d'information au hover (web)
- **Bouton "View All"** : même problème que SeriesScreen

### `SearchScreen.tsx` (281 lignes)
- ✅ **Search input** : utilise désormais le composant `SearchBar` réutilisable
- **Pas de search history** / recherches récentes
- **Pas de trending/popular searches** quand input vide
- **Pas de search suggestions** / autocomplete
- **Semantic search toggle** : seulement visible si `debouncedQuery.length > 10` — pas découvrable
- **MoodPicker** : bon concept mais visuellement basique (chips), pourrait être plus visuel avec des icons/emojis
- **Discover sections** : bon contenu mais headers basiques, pas de "see all" link
- **ShowCard** : pas de quick-add button, pas de rating affiché, pas de genre tags
- **Pas de filtre** type (TV/Movie) sur les résultats de recherche
- **Pas de vue grid** sur les résultats de recherche (toujours liste)

### `NewsScreen.tsx` (106 lignes)
- **Filter toggle** : basique `TouchableOpacity` avec toggle icon — pourrait être un `Switch` ou segmented control
- **NewsCard** : pas de category badge, pas de source name, pas de read time estimate
- **Desktop grid** : seulement 2 colonnes — pourrait être 3 sur très large écran
- **Pas de bookmark/save** d'articles
- **Pas de pull-to-refresh** sur le toggle filter (seulement sur la liste)
- **Pas de catégorie/filter** par type de news
- **Pas de temps de lecture** estimé

### `ProfileScreen.tsx` (346 lignes — dépasse la limite de 300)
- **Pas de cover/banner** image en haut du profil (pattern standard moderne)
- **Avatar section** : basique, pas de decoration, pas de level/badge
- **Stats grid** : `StatCard` très simple, pas d'animation au chargement, pas de comparison (vs semaine dernière)
- **Year in Review modal** : bon contenu mais `Modal` plein écran basique, pas de carousel/swipe entre les sections
- **Logout button** : plain bouton rouge en bas — pourrait être dans le menu ou un bottom sheet
- **ProfileMenuButton** : dropdown modal basique — pourrait être un bottom sheet sur mobile (plus ergonomique)
- **FavoriteCarousel** : bon concept mais pourrait avoir un mode "voir tout" → écran dédié
- **Pas de public profile preview** depuis son propre profil
- **Pas de streak calendar** / heatmap (GitHub-style) pour visualiser la régularité
- **RecentActivity** : basique, pas de filtre par type
- **AiInsights** : bon composant, visuellement basique

## 3. Pages détail

### `ShowDetailScreen.tsx` (786 lignes — dépasse largement la limite de 300)
- ✅ **Hero image** : utilise désormais le **backdrop** TMDB (16:9) via `HeroImage` avec gradient overlay

- **Genres** : basic chips `bg-surface rounded-full` — OK mais pas d'interaction (tap pour filtrer)
- ✅ **AI tags** : bon feature, visuellement OK
- **Cast/Crew** : `CastMemberCard` dupliqué entre ShowDetail et EpisodeDetail — **DRY violation**
- **Similar shows** : basic horizontal scroll, pas de rating/genre info sur les cards
- **Comments CTA** : deux layouts différents selon le count → inconsistent
- ✅ **FixedTrackingButton** : haptic feedback ajouté sur toggle watched/dropped
- **Pas de share button** sur ShowDetail (seulement sur EpisodeDetail)
- **Desktop layout** : side-by-side basique, pas de sticky sidebar pour les infos
- ✅ **Tab system** : sections tabbed (Overview / Episodes / Cast / Comments) via `SegmentedControl` sur mobile
- **EpisodeGrid** : `LazyEpisodeGrid` bon concept mais toggles basiques
- **Pas de "next episode to watch"** card en évidence pour les shows en cours
- **Pas de countdown** pour les shows avec prochain épisode

### `EpisodeDetailScreen.tsx` (550 lignes — dépasse la limite de 300)
- **Still image** : OK mais gradient overlay seulement sur mobile (`!isDesktopWeb`)
- **Season/Episode indicator** : bon design pill
- **Navigation bar** : bon pattern prev/next/show
- **Comments preview** : basique, pas de reaction counts, pas de preview des replies
- **AI Summary** : bon feature, visuellement basique
- ✅ **Rating** : `RatingStars` avec haptic feedback sur star press
- **Cast** : `CastMemberCard` dupliqué (même composant que ShowDetail)
- **Pas de "watched by X% of community"** stat
- **Pas de trivia/behind the scenes**
- **Pas de lien direct** vers la saison complète
- **Share button** : OK mais pourrait partager un deep link web

## 6. Design System — Problèmes transversaux

### Inconsistances
✅ **3 composants tabs différents** → unifié en `SegmentedControl`
✅ **3 variantes de cards liste** → unifié en `MediaRow` (ShowCard utilise MediaRow en interne)
- **`bg-muted` vs `bg-surface`** utilisé interchangeablement selon les écrans
- **`text-white` vs `text-background`** pour le texte sur primary (inconsistant)

### Manques
✅ **Système d'animation** : `react-native-reanimated` intégré (Snackbar, EmptyState, UnwatchedEpisodeRow swipe)
✅ **Haptic feedback** : système via `utils/haptics.ts` (tab press, mark watched, rating, swipe, long-press)
- **Pas de shimmer** sur les skeletons
- **Pas d'illustrations** pour empty states
- **Pas de Lottie** pour les animations de célébration/success
- **Pas de dark mode toggle** accessible depuis la main UI (seulement dans Profile > Appearance)
- **Pas de système de toast** différencié (snackbar fait tout)
- **Typography** : seulement Outfit, pas de hierarchy claire (H1/H2/body/caption)

### Couleurs
- Palette limitée : pas d'accent color, pas de warning color
- `surfaceLight` peu utilisé pour la profondeur
- Pas de gradient definitions

### Responsive/Web
- Un seul breakpoint `768px` — pas de `lg` (1024), `xl` (1280)
- DesktopSidebar basique
- Pas de hover states sur la plupart des éléments interactifs
- Pas de keyboard shortcuts
- Pas de cursor variants (`pointer` sur boutons web)
